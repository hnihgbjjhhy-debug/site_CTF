
## **Volatility**

### Описание

**Volatility** — это открытый фреймворк для анализа оперативной памяти (memory forensics), предназначенный для извлечения цифровых артефактов из дампов RAM. Он позволяет анализировать состояние системы на момент захвата памяти, выявлять malware, rootkits, инъекции процессов и другие угрозы, которые не оставляют следов на диске. Volatility работает независимо от исследуемой системы, поддерживает Windows, Linux, macOS и Android. Это один из самых популярных инструментов в цифровой криминалистике, incident response и malware analysis. Фреймворк написан на Python, модульный (плагиновая архитектура) и активно используется правоохранительными органами, военными и коммерческими исследователями.

### Версии Volatility (на начало 2026 года)

- **Volatility 2** (legacy): Старая ветка (последняя версия 2.6.x), устаревшая и заархивированная на GitHub. Поддержка прекращена в 2025 году после достижения паритета с Volatility 3. Требует профилей для ОС, работает на Python 2/3, но медленнее и менее гибкая.
- **Volatility 3** (текущая): Полная переписанная версия (с 2019 года), на Python 3.8+. Автоматическое определение профиля/символов, улучшенная производительность, поддержка современных ОС. Последняя стабильная версия — **2.26.2** (от сентября 2025 года, скомпилирована в октябре 2025 для некоторых инструментов). Volatility 3 достигла feature parity с Volatility 2 в 2025 году и теперь является основной поддерживаемой веткой.

### Основные инструменты

Volatility 3 — CLI-утилита (vol или vol.py) с плагинами. Основные команды/плагины (примеры для Windows; аналогично для linux/mac):

- **windows.info** / **linux.info** / **mac.info** — Базовая информация о системе (ОС, версия, архитектура, KDBG).
- **windows.pslist** / **windows.pstree** — Список процессов и дерево процессов.
- **windows.netscan** — Сканирование сетевых соединений.
- **windows.dumpfiles** / **windows.procdump** — Дамп файлов/процессов из памяти.
- **windows.malfind** — Поиск инъекций в процессы (malware).
- **windows.cmdline** — Командные строки процессов.
- **windows.dlllist** — Загруженные DLL.
- **windows.hashdump** / **windows.lsadump** — Дамп хэшей паролей и LSA secrets.
- **volshell** — Интерактивная оболочка для исследования памяти.
- Другие: **windows.memmap**, **linux.pslist**, **mac.pslist**, **windows.registry** и т.д.

Плагины разделены по ОС (windows._, linux._, mac.*). Поддержка символов: автоматическая для Windows, ручная для Linux/Mac (через dwarf2json).

## **Wireshark**
### Описание

**Wireshark** — это бесплатный и открытый (open-source) анализатор сетевых протоколов (network protocol analyzer), считающийся мировым стандартом для захвата и анализа сетевого трафика. Он позволяет в реальном времени захватывать пакеты данных из сети, детально их разбирать и анализировать на уровне протоколов. Wireshark используется для диагностики сетевых проблем, отладки протоколов, обучения, анализа безопасности (например, в forensics и CTF) и разработки ПО.

Первоначально проект назывался Ethereal (с 1998 года, автор Gerald Combs), но в 2006 году был переименован в Wireshark из-за проблем с торговой маркой. Инструмент поддерживает тысячи протоколов (более 3000), работает на Windows, macOS, Linux и других UNIX-подобных системах. Он использует библиотеку pcap (libpcap/WinPcap/Npcap) для захвата пакетов.

Wireshark отображает трафик в удобном интерфейсе с цветовой подсветкой, фильтрами и статистикой. Это незаменимый инструмент для сетевых администраторов, пентестеров и разработчиков.

### Версии Wireshark (на начало 2026 года)

Wireshark выпускается в двух основных ветках:

- **Stable (стабильная)**: Рекомендуется для большинства пользователей. На январь 2026 года последняя стабильная версия — **4.6.x** (например, 4.6.0 или новее в серии 4.6).
- **Development (разрабатываемая)**: Более новые фичи, но менее стабильная (обычно версия 4.7.x или выше).

Обновления выходят регулярно (каждые 1–2 месяца). Проект поддерживается Wireshark Foundation. Поддержка старых ОС постепенно прекращается (например, Windows 10 поддерживается только до 4.2, 32-bit Windows — до 3.6).

### Основные инструменты

Wireshark имеет графический интерфейс (на базе Qt) и командную строку (tshark). Ключевые компоненты и функции:

- **Packet Capture**: Захват пакетов в реальном времени с различных интерфейсов (Ethernet, Wi-Fi, Bluetooth, USB и т.д.).
- **Display Filters**: Мощные фильтры для поиска (например, http contains "password" или ip.addr == 192.168.1.1).
- **Packet List/Details/Bytes**: Три панели — список пакетов, детальный разбор и hex-вид.
- **Follow Stream**: Сборка потоков (TCP/UDP/HTTP) для чтения содержимого (например, весь HTTP-запрос).
- **Statistics**: Статистика протоколов, графики потоков (Flow Graph), IO Graphs, Conversations/Endpoints.
- **Expert Information**: Автоматическое выявление проблем (ошибки, предупреждения).
- **Dissectors**: Более 3000 диссекторов протоколов для глубокого анализа.
- **Command-line tools**: tshark (аналог Wireshark в терминале), dumpcap (захват), editcap/mergecap (редактирование файлов), capinfos и т.д.
- **Export/Import**: Поддержка множества форматов (pcapng, pcap и другие).

Дополнительно: цветовые правила, декодирование зашифрованного трафика (с ключами), VoIP-анализ и т.д.

## **Autopsy**
### Описание

**Autopsy** — это открытая (open-source) цифровая forensic-платформа с графическим интерфейсом, построенная на базе **The Sleuth Kit** (TSK) и других инструментах. Она предназначена для анализа дисков, образов (disk images), смартфонов и локальных файлов. Autopsy используется правоохранительными органами, военными, корпоративными экспертами и частными исследователями для расследования инцидентов: восстановления удалённых файлов, анализа timeline событий, поиска артефактов (веб, реестр, email) и т.д. Инструмент поддерживает многопользовательский режим (multi-user cases), параллельную обработку на нескольких ядрах, модульную архитектуру (ingest modules) и отчёты в HTML/Excel. Autopsy бесплатный, кросс-платформенный (Windows, Linux, macOS) и активно развивается сообществом.

### Версии Autopsy (на начало 2026 года)

Текущая стабильная версия — **4.22.1** (выпущена недавно, фиксит баг с Excel и откатывает версию TSK для совместимости). Основная ветка — **Autopsy 4.x** (с 2016 года). Предыдущие значимые: 4.22.0 (добавлена поддержка BitLocker и совместимость с Cyber Triage). Обновления выходят регулярно, проект ведётся на GitHub (sleuthkit/autopsy). Нет разделения на "free/pro" — полностью open-source под Apache License 2.0, но есть коммерческая поддержка от Sleuth Kit Labs.

### Основные инструменты

Autopsy — модульная система с "ingest modules" (автоматическая обработка при добавлении источника данных). Ключевые функции и модули:

- **Timeline Analysis** — Графический таймлайн событий (файлы, реестр, логи).
- **Keyword Search** — Индексированный поиск по тексту, регулярным выражениям (с использованием Solr/Lucene).
- **File Analysis** — Просмотр файловой системы, восстановление удалённых файлов, hash lookup (NSRL, известные bad/good хэши).
- **Interesting Files** — Автоматическое выделение подозрительных файлов/папок по имени, пути.
- **Media View** — Галерея изображений/видео, извлечение EXIF.
- **Web Artifacts** — Анализ браузеров (история, куки, скачанные файлы).
- **Registry Analysis** — Парсинг Windows реестра (USB устройства, запущенные программы).
- **Email Parsing** — Извлечение email и вложений.
- **Android Support** — Анализ SMS, звонков, контактов, приложений.
- **Reports** — Генерация отчётов в HTML, Excel, Bodyfile.
- **Multi-Core Processing** — Параллельная обработка для быстрых результатов.
- **Extensibility** — Поддержка плагинов (Python/Java), third-party modules.

Поддерживает форматы: raw/dd, E01, NTFS, FAT, EXT, HFS+ и др.

## **Binwalk**
### Описание

**Binwalk** — это мощный open-source инструмент для анализа, реверс-инжиниринга и извлечения данных из бинарных файлов, в первую очередь образов прошивок (firmware) устройств (роутеры, IoT, embedded systems). Он сканирует файлы на наличие встроенных сигнатур файлов, архивов, файловых систем, исполняемого кода и т.д. Binwalk использует библиотеку libmagic с кастомными сигнатурами для firmware, поддерживает entropy-анализ для выявления сжатых/зашифрованных участков и автоматическое извлечение. Широко применяется в цифровой криминалистике (forensics), пентестинге, reverse engineering и анализе уязвимостей firmware. Инструмент CLI-based, работает на Linux/Windows/macOS, интегрируется с другими tools (например, dd, unzip).

### Версии Binwalk (на начало 2026 года)

Классическая версия на Python (оригинал от Craig Heffner/ReFirmLabs) — последняя стабильная **v2.3.4** (с фиксами безопасности, такими как предотвращение directory traversal). Есть переписанная версия на Rust (от ReFirmLabs) для большей скорости и точности, но она всё ещё в развитии (не имеет явного номера выше 3.x в релизах). Некоторые форки упоминают v3.1.0 (улучшения в 2025). Активно поддерживается на GitHub, обновления включают новые сигнатуры и плагины. В Kali Linux предустановлена актуальная версия.

### Основные инструменты

Binwalk — это в основном CLI-утилита с опциями:

- **Signature scanning** (-B): Поиск распространённых сигнатур файлов (gzip, zip, squashfs, uImage, TRX и т.д.).
- **Extraction** (-e, -M, --dd): Автоматическое/рекурсивное извлечение встроенных файлов и архивов.
- **Entropy analysis** (-E): Построение графика энтропии для выявления сжатых/зашифрованных данных.
- **Disassembly** (-A, -Y): Поиск опкодов исполняемого кода и определение архитектуры CPU (с Capstone).
- **Custom signatures** (-m, -R): Использование своих magic-файлов или raw-байт.
- **Matryoshka** (-Me): Рекурсивное извлечение вложенных архивов.
- **Дополнительно**: Фильтры (--include/--exclude), статус-сервер, интеграция с внешними экстракторами (unzip, sasquatch для squashfs).

Примеры команд:

- binwalk firmware.bin — базовый скан.
- binwalk -Me firmware.bin — полное рекурсивное извлечение.
- binwalk -E firmware.bin — энтропия.

Поддерживает плагины и интеграцию (например, с IDA Pro).
